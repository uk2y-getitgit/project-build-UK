# Safety-Direct: 현장 맞춤형 안전관리계획서 자동화 시스템 — 프로젝트 로드맵

---

## 1. 프로젝트 컨텍스트 및 비전

**핵심 가치:** 법령 개정에 즉각 대응하는 고품질 안전관리계획서의 100% 자동 생성 체계 구축

- 수작업 최소화
- 법적 대응력 강화
- PDF + HWPX 하이브리드 출력 지원

모든 코드 작성과 설계 제안은 이 비전을 기준으로 진행한다.

---

## 2. 하이브리드 문서 생성 전략

### 문서 구성 방식

| 파트 | 대상 | 생성 방식 |
|------|------|----------|
| 가변 파트 | 1장, 2장 | 사용자 입력 데이터 기반 실시간 렌더링 → HWPX 생성 |
| 고정 파트 | 3장 부록 | 사용자 업로드 원본 PDF 단순 병합(Merge) → 서버 부하 최소화 |

### 출력 원칙

- **전체본:** PDF로 제공
- **선별 출력:** 조직도 등 수정이 필요한 특정 양식만 HWPX 다운로드 허용

### 데이터 관리

- 모든 데이터는 **JSON 기반**으로 아카이빙
- '공사명' 등 공통 데이터는 모든 페이지에 자동 동기화되는 **Single Source** 로직 구현

---

## 3. 에이전트 기반 개발 워크플로우

4단계 에이전트 협업 체계를 통해 자동화 파이프라인 구동. (상세 정의: `AGENTS.md` 참조)

| 에이전트 | 역할 |
|---------|------|
| Agent A (분석가) | HWPX XML 분석 및 누름틀 변수 추출 |
| Agent B (설계자) | HTML 입력 UI 구성 및 데이터 포맷팅 로직 구현 |
| Agent C (QA) | Mock 데이터 테스트 및 레이아웃 검수 |
| Agent D (승인자) | 법적 필수 항목 체크 및 최종 배포 결정 |

모든 작업은 `CLAUDE.md` 및 `.claude/skills/` 가이드라인 기반으로 협업한다.

---

## 4. 운영 정책 및 리스크 관리

### 4-1. 버전 관리
- 단순 버전 번호가 아닌 **적용 법령 개정 시점**을 기준으로 템플릿 아카이빙

### 4-2. 데이터 격리 및 자동 삭제
- 사용자별 독립된 보관 환경 제공
- 문서 생성 후 **30일 경과 시 Cron Job으로 자동 영구 삭제**

### 4-3. 타임아웃 방지
- 수백 장 문서를 한 번에 렌더링하지 않음
- **장(Chapter)별 분할 생성 아키텍처** 도입

### 4-4. 규격 유지
- A4, A3 등 다양한 규격 PDF 병합 시 **원본 Scale 유지**

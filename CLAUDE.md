# Safety-Direct 프로젝트 지침

> 이 파일은 Claude Code가 이 프로젝트에서 작업할 때 따르는 규칙입니다.
> 새 환경에서 작업 시작 전 반드시 이 파일을 확인하세요.

---

## 1. 핵심 목표

- 제공된 HWPX 양식의 레이아웃과 서식을 100% 준수한다.
- HTML UI와 HWPX 필드 간의 매핑 정확도를 최우선으로 한다.

## 2. 데이터 포맷팅 표준 (반드시 준수)

- **날짜:** 반드시 `YYYY-MM-DD` 형식을 유지한다.
- **숫자/금액:** 천 단위 구분 기호(`,`)를 자동 적용한다.
- **HWPX 정렬:** 표 내부 데이터는 중앙 또는 좌측 정렬을 통일하고, 글자 크기는 양식의 기본값을 유지하되 줄바꿈이 균일하게 적용되도록 제어한다.

## 3. 코드 최적화 지침

- 토큰 소모를 최소화하기 위해 중복 코드를 제거하고 모듈화한다.
- 불필요한 라이브러리 호출이나 반복되는 로직은 공통 함수(utils)로 통합한다.

## 4. UI/UX 디자인 원칙

- 모든 입력 페이지는 통일된 테마를 사용한다.
- 입력창(Input), 달력(Picker), 버튼의 크기를 충분히 확보하여 시인성을 높인다.
- 반응형 배치를 통해 입력 요소가 겹치지 않게 구성한다.

## 5. 언어 규칙 (전역 CLAUDE.md와 동일)

- 모든 응답은 반드시 **한국어**로 작성한다.
- 코드 주석도 한국어로 작성한다.
- 커밋 메시지는 한국어로 작성한다. (형식: `타입: 변경 내용`)
